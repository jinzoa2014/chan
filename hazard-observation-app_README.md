# Hazard Observation App

A company-wide safety reporting application built to replace large, image-heavy Excel files that were slow and laggy to open. Rebuilt as a fast web app with a rectification approval system so hazards are tracked from report to resolution.

## The Problem

Hazard observations were stored in large Excel files packed with images. Opening these datasets was slow and laggy, making company-wide reporting painful and assessment difficult. There was no structured way to track whether a reported hazard had actually been rectified.

## What I Built

- **Fast web-based hazard reporting** that handles images without the lag of large spreadsheets.
- **Rectification approval system** — each hazard observation moves through a review/approval flow during rectification, so nothing is closed without sign-off.
- **Better assessment views** for reviewing hazards across the company.

## Tech Stack

- **Backend:** Python
- **Frontend:** React
- **Database:** SQLite

## Outcome

Faster, smoother safety reporting for the whole company, with a clear approval trail for hazard rectification — replacing a slow Excel-based process.
