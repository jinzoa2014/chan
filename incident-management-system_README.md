# Incident Management System

A department-wide incident monitoring and approval system that replaced flat, hard-to-search Excel logs with a proper web application — complete with a multi-step approval workflow, automatic email notifications, and an image-rich dashboard.

## The Problem

Incidents were being tracked in shared Excel files. They were slow to search, easy to overwrite, had no approval trail, and couldn't show supporting photos in any useful way. There was no automatic notification when something needed sign-off.

## What I Built

- **Incident logging** with supporting images, so reports are visual rather than rows of text.
- **Multi-step approval workflow** — incidents move through review and approval stages with a clear status at each point.
- **Automatic email notifications** triggered after approval, so the right people are informed without manual follow-up.
- **Dashboard view** presenting incident data and images in a clean layout instead of raw spreadsheet cells.

## Tech Stack

- **Backend:** Python
- **Frontend:** HTML
- **Database:** SQLite
- **Notifications:** automated email on approval events

## Outcome

Faster incident tracking, a reliable approval trail, and automatic notifications — removing manual chasing and the risk of overwritten records.
